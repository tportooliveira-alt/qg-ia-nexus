# Checklist de Variáveis de Ambiente (Nexus Hub)
Use este arquivo como guia. Não coloque valores reais aqui e não commite `.env`.

## 1) qg-ia-nexus/.env (backend)
- PORT=3005
- QG_AUTH_TOKEN=<<token único>>
- GEMINI_API_KEY=...
- GROQ_API_KEY=...
- ANTHROPIC_API_KEY=...
- OPENAI_API_KEY=...
- CEREBRAS_API_KEY=...
- DEEPSEEK_API_KEY=...
- SUPABASE_URL=...
- SUPABASE_SERVICE_KEY=...
- DB_HOST=...
- DB_USER=...
- DB_PASS=...
- DB_NAME=...
- FABRICA_API_URL=https://fabrica-ia-api.onrender.com
- FABRICA_API_KEY=...
- STITCH_BASE_URL=https://stitch.googleapis.com/mcp
- STITCH_API_KEY=...   (preencher)
- ENABLE_WHATSAPP=false
- TOKEN_VOLUME=normal
- DOMAIN_CONFIDENCE_THRESHOLD=0.25

## 2) qg-ia-nexus/apps/web/.env.local (frontend Vite)
- VITE_BACKEND_URL=https://ideiatoapp.me   (ou http://localhost:3005 em dev)
- VITE_QG_TOKEN=<<mesmo QG_AUTH_TOKEN>>

## 3) fabrica-ia-api/.env
- PORT= (se usar)
- SUPABASE_URL=...
- SUPABASE_ANON_KEY=...
- FABRICA_AUTH_TOKEN=...

## 4) frigogest-2026/.env
- Variáveis conforme stack (Firebase/Supabase/API keys).

## 5) agromacro/.env
- Endpoints/storage/auth conforme o projeto.

## 6) gestao-fazenda-antares/.env
- React/Firebase/Supabase conforme o código.

## 7) IdeaOrganizer/.env
- Se usar backend ou serviços externos.

### Observações
- Guarde valores reais só nos `.env` locais.
- Copie este checklist e substitua “...” nos arquivos corretos; não commite os `.env`.
- Para produção (Hostinger), use os mesmos valores do backend e defina `VITE_BACKEND_URL` para o domínio público.
